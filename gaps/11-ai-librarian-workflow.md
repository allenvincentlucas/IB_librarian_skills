# Gap 11: Using Claude / AI as a Librarian Workflow Tool

*New research added for the IB Librarian Skills website, July 2026. In-text citations follow MLA 9th edition, with a bracketed number [n] alongside each for quick lookup against the numbered Works Cited list at the end.*

## 1. The reframe: this is library science applied to a new format

A useful framing for staff who are skeptical of AI tools: helping people evaluate provenance, judge source authority, and decide what to trust are questions librarians have answered for over a century, just applied to a new kind of output (Public Libraries Online [3]). That reframe matters when introducing Claude to colleagues who see AI adoption as a separate "tech skill" rather than an extension of the professional judgment they already have.

## 2. Where AI genuinely saves time in library operations, and where it doesn't

A practitioner walkthrough of AI tool rollout in libraries identifies staff resistance and workflow disruption with existing systems as the two most common failure points, and stresses that a tool needs a clear, narrow operational brief — what it should and shouldn't do — rather than being deployed as an open-ended assistant (Kumar [2]). In practice this means writing a specific, reusable prompt for a specific task (a book-display blurb, a Titlewave list description, a parent-newsletter summary) rather than treating every use as a one-off conversation.

Concrete staff-level applications worth piloting first: drafting first-pass marketing copy for library events and reading challenges (then editing for voice and accuracy), experimenting with AI-assisted subject-heading suggestions for a cataloging backlog (treated as a draft requiring professional verification, not a final answer), and drafting grant proposal language that a librarian then refines with real institutional context (Public Libraries Online [3]).

## 3. Prompt libraries: reusable templates beat one-off requests

Anthropic maintains a public library of ready-to-copy prompts, organized around a repeatable pattern: describe the outcome wanted, not the exact steps, and let the tool figure out the path (Anthropic, "Prompt library" [1]). The same logic applies directly to library work — a workshop or PD session for staff is far more useful if it produces a shared bank of tested prompts (an event-copy template, a collection-audit summary template, a parent-communication template) than if it just demonstrates the tool once and moves on.

A specifically librarian-facing example: a school-librarian newsletter built a themed set of AI prompts and templates for reader-advisory lists, themed book displays, and event promotion, paired with instructions for turning that AI output into finished graphics in a design tool — a concrete pattern worth adapting into this site's own PD training materials (The AI School Librarians Newsletter [4]). A broader K-12 prompt library aimed at lesson planning and administrative tasks across multiple AI chatbots is useful as a comparison point for what a non-library-specific version of the same idea looks like (AI for Education [5]).

## Bottom line

Three deliverables would close this gap, and they connect directly to the existing 21-tutorial roadmap and five-module staff PD workshop already built for this project: (1) a shared, versioned prompt library specific to library operations (event copy, collection audits, committee reports, parent communications), rather than leaving each staff member to reinvent prompts individually, (2) an explicit "draft, don't finalize" rule for any AI-assisted cataloging or subject-heading work, matching the verification standard already used for AI-assisted research with students, and (3) framing AI-literacy PD for library staff around the reframe in source [3] — this is an extension of professional judgment they already hold, not a separate technical skill to be anxious about.
